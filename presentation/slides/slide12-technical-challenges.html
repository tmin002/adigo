What We Learned

Key insights from development challenges

1. UWB Implementation Challenges

Original Plan: Precise location sharing using UWB technology for crowded places.

Key Obstacle: UWB devices need ID handshake via Bluetooth, but heavy interference in crowded areas prevented proper connection.

Decision: Considered WebSocket-based key sharing but too many edge cases. Focused on core features instead.

2. Planning is Critical

Issue: Started without coding conventions or API definitions.

Result: As the service scaled, error handling became difficult and code structure became unclear.

3. Optimization Matters

Battery Life

5-second location updates drained battery heavily. Fixed by sending only when device moves using motion sensor.

Smart Data Storage

Redis was perfect for location data - much more efficient than saving to disk.

Async Programming

Used RxJava for non-blocking operations. Defined clear exception types for easier error handling.
